     #1. ¿Qué es una Base de Datos Relacional?

Es una forma de almacenar información usando tablas.
Los datos se organizan en filas y columnas.
Las tablas pueden relacionarse entre sí.
Esto permite tener la información ordenada y fácil de consultar.

#2. ¿Qué es una Tabla en una Base de Datos Relacional?

Es una estructura donde se guardan los datos.
Tiene columnas y renglones, como Excel.
Cada tabla guarda información de un mismo tipo.
Ejemplo: usuarios, productos o clientes.

#3. ¿Qué es una relación en una Base de Datos Relacional?

Es la conexión entre dos o más tablas.
Sirve para compartir información entre ellas.
Evita repetir los mismos datos.
Se hace usando claves.

#4. ¿Qué representa una columna?

Una columna representa un tipo de dato.
Describe una característica específica.
Ejemplos: nombre, edad, correo.
Todas las filas usan las mismas columnas.

#5. ¿Qué representa un renglón?

Un renglón representa un registro completo.
Contiene todos los datos de una persona u objeto.
Cada fila es diferente.
Ejemplo: los datos de un usuario.

#6. ¿Qué es un motor de base de datos relacional?

Es el programa que maneja la base de datos.
Permite guardar, consultar y modificar datos.
Se encarga de la seguridad y organización.
Es esencial para que la base funcione.

#7. ¿Cuáles son los motores relacionales más populares?

Algunos motores muy usados son MySQL y PostgreSQL.
También Oracle y SQL Server.
MariaDB es otra opción común.
Se usan en sistemas y aplicaciones reales.

#8. ¿Cuál es el motor ligero que no necesita servidor?

SQLite es un motor ligero.
No necesita instalar un servidor.
Guarda la base en un solo archivo.
Es ideal para proyectos pequeños.

#9. ¿Qué significa SQL?

SQL significa Structured Query Language.
Es un lenguaje para trabajar con bases de datos.
Sirve para consultar y modificar información.
Se usa en bases de datos relacionales.

#10. ¿Cuáles son los elementos de un sistema de información?

La base de datos donde se guarda la información.
El motor que la administra.
El lenguaje SQL para consultarla.
Los usuarios y el sistema que la usa.
